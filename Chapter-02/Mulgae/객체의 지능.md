여기서 말하는 “객체의 지능”은 처음 들으면 좀 애매한 표현인데, 사실 어려운 개념은 아니야.

핵심은 이거야:

> **객체의 지능 = 스스로 판단하고 결정할 수 있는 능력**

조금 더 풀어서 말하면

👉 “외부가 시키는 대로 움직이는 존재”가 아니라

👉 “자기 책임 범위 안에서 스스로 판단하고 행동하는 능력”이야.

---

# 1. 지능이 낮은 객체 vs 높은 객체

## ❌ 지능이 낮은 객체 (상태 중심)

```
struct Account {
    var balance: Int
    var isLocked: Bool
}

final class AccountService {
    func withdraw(account: inout Account, amount: Int) {
        if account.isLocked {
            print("잠김")
            return
        }

        if account.balance >= amount {
            account.balance -= amount
        }
    }
}
```

### 특징

- Account는 그냥 데이터 덩어리
- “출금 가능한지”를 **스스로 판단하지 못함**
- 외부(AccountService)가 다 판단함

👉 이런 객체는 “지능이 없다”고 표현해

(정확히는 **판단 능력이 외부에 있다**는 뜻)

---

## ✅ 지능이 높은 객체 (행동 중심)

```
enum AccountError: Error {
    case locked
    case insufficientBalance
}

final class Account {
    private var balance: Int
    private var isLocked: Bool

    init(balance: Int, isLocked: Bool) {
        self.balance = balance
        self.isLocked = isLocked
    }

    func withdraw(_ amount: Int) throws {
        if isLocked {
            throw AccountError.locked
        }

        if balance < amount {
            throw AccountError.insufficientBalance
        }

        balance -= amount
    }
}
```

### 특징

- “출금 가능한지”를 **스스로 판단함**
- 외부는 그냥 “출금해줘”라고 요청만 함

```
try account.withdraw(1000)
```

👉 이 객체는 자기 상태 + 규칙을 함께 가지고 있음

👉 그래서 “지능이 높다”고 표현하는 거야

---

# 2. 지능 = 판단 + 책임

조금 더 정확하게 정의해보면:

> **객체의 지능 = 자신의 상태를 기반으로 올바른 행동을 선택할 수 있는 능력**

즉, 2가지가 합쳐진 개념이야:

### 1) 상태를 알고 있다

- balance
- isLocked

### 2) 상태를 해석할 수 있다 (이게 핵심)

- “지금 출금 가능한 상태인가?”
- “이 요청은 허용해야 하는가?”

👉 이 “해석 + 판단” 능력이 바로 지능

---

# 3. 왜 캡슐화 → 자율성 → 지능으로 이어질까?

흐름을 보면 이렇게 연결돼.

### 1) 캡슐화

- 상태를 외부에서 못 건드림
- 규칙을 내부에 숨김

↓

### 2) 자율성 증가

- 외부가 간섭 못함
- 객체가 스스로 처리해야 함

↓

### 3) 지능 증가

- 판단 로직이 객체 안으로 들어옴
- 객체가 “생각하는 존재”가 됨

---

# 4. 직관적인 비유

## 🧱 지능 낮은 객체 (상태 중심)

- 엑셀 셀
- 값만 있음
- 계산은 사람이 함

## 🧑‍💼 지능 높은 객체 (행동 중심)

- 직원
- 규칙을 알고 있음
- 요청 받으면 판단하고 처리함

```
상태 중심:   "잔액이 얼마야?" → 내가 계산해서 빼자
행동 중심:   "1000원 출금해줘" → 객체가 알아서 판단
```

---

# 5. 지능이 높아지면 생기는 변화

## (1) 코드가 더 안전해짐

잘못된 상태를 외부에서 못 만듦

## (2) 변경이 쉬워짐

규칙이 한 곳(객체 내부)에 모여 있음

## (3) 협력이 자연스러워짐

객체끼리 “요청”을 주고받음

```
order.pay() // 주문이 책임짐
product.reserve() // 상품이 책임짐
```

---

# 6. 중요한 오해 하나

여기서 “지능이 높다”는 건

👉 **객체가 똑똑한 AI처럼 복잡해진다는 뜻이 아님**

오히려 반대야.

- 각 객체는 **자기 책임만** 똑똑하게 처리하면 된다
- 전체 시스템은 **여러 객체의 협력으로 지능이 만들어진다**

👉 “작게 똑똑한 객체들의 협력”이 목표야

---

# 한 줄 정리

> 객체의 지능이란
>
> **자기 상태를 바탕으로 스스로 판단하고 행동할 수 있는 능력**이다.
