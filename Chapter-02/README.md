# 📖 02. 이상한 나라의 객체

## 1. 객체를 구성하는 3요소

### 🟢 상태 (State)
객체는 상태를 가지며, 이 상태는 행동에 의해 변경될 수 있습니다.
* **필요성**: 과거의 복잡한 행동 이력을 기억할 필요 없이, **현재 상태**만으로 객체의 다음 동작을 결정할 수 있게 합니다.
* **구성**:
    * **프로퍼티 (Property - 정적)**: 객체의 구조적 특징을 정의합니다.
        - **속성 (Attribute)**: 객체 내부의 단순한 값.
        - **링크 (Link)**: 다른 객체와의 연결(참조).
    * **프로퍼티 값 (Property Value - 동적)**: 시간에 따라 변하는 실제 데이터값.

---

### 🔵 행동 (Behavior)
객체의 상태를 변경시키는 실질적인 활동입니다.
* **상태와의 관계**: 
    - 행동은 현재 상태에 의존적입니다. (ex. 연료가 없으면 주행할 수 없음)
    - 행동의 결과로 객체의 상태가 변화합니다.
* **협력과 부수 효과**:
    - 자신의 상태를 변경할 뿐만 아니라, 협력 중인 **타 객체에 메시지를 전송**하여 영향을 미칩니다.
* **상태 캡슐화**:
    - 객체는 상태를 외부에 노출하지 않고 **행동만 노출**합니다.
    - 송신자는 수신자의 상태를 직접 알 수 없으며, 오직 메시지를 통해 소통합니다.

---

### 🔴 식별자 (Identity)
객체와 값을 구분 짓는 가장 중요한 요소입니다.

| 구분 | 값 타입 (Value Type) | 객체 (Reference Type) |
| :--- | :--- | :--- |
| **핵심** | **동등성 (Equality)** | **동일성 (Identity)** |
| **기준** | 인스턴스가 달라도 상태가 같으면 같음 | 상태가 변해도 식별자가 같으면 같음 |
| **성질** | 불변성 (Immutability) | 가변성 (Mutability) |
| **Swift** | `struct`, `enum` | `class` |

> **💡 Note**: Swift는 `struct`와 `class`를 통해 이 구분을 언어 차원에서 명확히 지원합니다.

---

## 2. RDD: 행동이 상태를 결정한다
상태(데이터)를 먼저 설계하고 행동을 나중에 정하는 방식은 지양해야 합니다.

* **상태 우선 설계의 단점**:
    1. 내부 구현이 드러나 **캡슐화**가 깨집니다.
    2. 객체가 고립되어 원활한 **협력**이 어렵습니다.
    3. 특정 데이터 구조에 얽매여 **재사용성**이 낮아집니다.

---

## 3. 은유와 객체
객체지향 세계의 객체는 단순히 현실 세계를 모방하는 것이 아닙니다.

* **의인화**: 현실에선 수동적인 존재(ex. 전등)도 객체 세계에선 스스로 행동하는 능동적인 존재가 됩니다.
* **은유**: 현실 세계의 이름을 빌려오는 것은 객체의 의미를 더 쉽게 전달하기 위한 도구일 뿐입니다.
